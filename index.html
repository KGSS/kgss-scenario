Kerbin Geographic and Science Society Scenarios

for Kerbal Space Program

Download this project as a .zip file Download this project as a tar.gz file

Requirements

  • Kerbal Space Program version 0.17
  • Operating systems: Windows, Mac OSX

Downloading

Click on the .tar.gz or .zip buttons at the top of this page to download the pack.

Installation

When the download is complete, extract the contents and open the folder. You will be presented with the files and folders listed below, please read the information and follow the instructions for installation where necessary.

  • Plugins - Copy the contents of this folder to KSP/Plugins
  • banners - Copy the contents of this folder to KSP/saves/scenarios/banners
  • scenarios - Copy the contents of this folder to KSP/saves/scenarios
  • src - The plugin source is present for compliance with the Kerbal Space Program forum rules
  • README - A link to these instructions
  • COPYING - The licence the source is distributed under.

Running the Scenarios

  1. Run the Kerbal Space Program game
  2. At the start screen select "Start Game"
  3. At the next screen select "Scenarios"
  4. Select a scenario from the list.

Information about the scenarios included in this pack are detailed in the following sections.

Nanobots Awry

This scenario introduces random failures to your ships, in what is essentially the same as the sandbox mode of the game. The aim of this is to add an extra layer of difficulty, by requiring the player to design craft which are able to withstand failures, for the duration of common objectives such as landing on the Mun.

Features

  • Random exploding parts
  • Random part disable and enable
  • Probability of failure scales with the size of the craft

Planned Features

  • Increased probability of explosion for struts and aerofoils
  • No exploding command pods
  • Engine failure wherein it exhibits on/off cycle behaviour
  • Modifying the endpoints of fuel lines
  • GUI to indicate when and what type of failure has occurred
  • Balancing the difficulty
  • User modifiable failure rates
